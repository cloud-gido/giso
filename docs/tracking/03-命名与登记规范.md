# 命名与登记规范

> 对应大同「规范管理：参数管理 / 事件管理 / 元素管理 / 页面管理」。
> **运行时权威**：PostgreSQL 库 `giso`（管理台读写）；**Git 镜像**：`schema/*.yaml`（codegen + 审计）。**先登记、后开发**。

---

## 1. 四张注册表

| 文件 | 内容 | 唯一键 |
|---|---|---|
| `schema/params.yaml` | 参数池：全局参数字典 | `key` |
| `schema/pages.yaml` | 页面池：所有页面 | `pgid` |
| `schema/elements.yaml` | 元素池：所有可曝光/可点击元素 | `eid` |
| `schema/biz_events.yaml` | 业务事件字典 | `code` |

## 2. 命名规则

### 页面 pgid

`{业务域}_{页面}`，业务域前缀：

| 前缀 | 业务线 |
|---|---|
| （无前缀，全局） | 首页、搜索、我的、登录等公共页面 |
| `video_` | 长视频 |
| `bet_` | 体育博彩 |
| `pm_` | 预测市场 |
| `news_` | 资讯 |

示例：`home`、`video_detail`、`bet_match_detail`、`pm_market_detail`、`news_article`。

### 元素 eid

`{名词}_{类型后缀}`，类型后缀收敛为：`_card`（卡片）、`_btn`（按钮）、`_tab`（页签）、`_banner`、`_entry`（入口）、`_item`（列表项）、`_bar`（操作条）。

**元素是全局的，不绑定页面**。`video_card` 在推荐页和搜索页是同一个 eid，页面差异由 `pgid` 表达，位置差异由 `mod`/`pos` 表达。禁止出现 `home_video_card` 这种把页面编进元素名的命名。

### 参数 key

见协议文档 §4。补充：参数描述里必须写清**取值规则**（如 `pos: 0-based，相对所在模块`），口径歧义在登记时消灭。

### 业务事件 code

`{域}_{对象}_{动作}`，动词用一般现在时：`video_play_start`、`bet_placed`、`pm_order_filled`、`deposit_completed`。客户端意图与服务端事实分开命名（`bet_submit` vs `bet_placed`）。

## 3. 登记与变更流程

### 3.1 生产环境（PostgreSQL + 管理台）

```
业务/编辑员提需求 ──► 管理台登记(pending 待审批) ──► 管理员批准 ──► live 生效
      ──► 可选 testing 联调 ──► 导出 YAML / PR ──► CI generate.py
```

| 状态 | 参与 /v1/track 校验 | 谁可写入 |
|------|---------------------|----------|
| **pending** | ❌ 待审批 | 编辑员提交；管理员可改 |
| draft / dev | ❌ | 管理员 |
| testing | ✅（测试 topic 口径） | 管理员发布 |
| **live** | ✅ | 管理员批准或发布 |
| deprecated | ❌（历史保留） | 管理员废弃 |

- **编辑员（editor）**：可新增/修改登记，保存后固定为 `pending`，**不直接生效**。
- **管理员（admin）**：在「待审批」或注册表行内 **批准** → `live`；可 **驳回** → `draft`。
- 审计：`registry_audit` 含 `approve` / `reject` 动作。

### 3.2 Git 标准路径（与 PR 评审）

```
业务/数据提需求 ──► 修改 schema/*.yaml 提 PR ──► 数据负责人 + 终端负责人 CR
      ──► 合并 ──► CI 生成双端常量 ──► 终端开发引用常量
      ──► 联调校验 ──► 上线
```

规则：

1. **PR 即登记单**：PR 描述里写明需求来源（任务链接）、统计口径、看数方式，对应大同「在 TAPD 上登记数据上报需求 + 自动关联」。
2. **删除/改义需走 deprecation**：参数和事件只增不改义；废弃字段标 `deprecated: true` 并保留至少两个版本，避免新老版本口径打架（对应大同版本控制：v1.0 / v2.0 结构体并存校验）。
3. **CI 强校验**：注册表 YAML 不合 schema（缺类型、缺负责人、命名违规、key 冲突）直接挂 CI。
4. **代码生成**：CI 从注册表生成 `sdk/web/src/generated.ts` 与 Android 常量类，业务代码禁止手写裸字符串 eid/pgid/参数 key。

## 4. 登记表字段定义

```yaml
# params.yaml 条目
- key: vid
  type: string          # string | int | float | bool(0/1) | object
  desc: 视频ID
  rule: 服务端视频主键，长视频正片/预告片通用
  owner: data-team
  since: "1.0"

# elements.yaml 条目
- eid: video_card
  desc: 视频卡片（封面+标题+CP信息）
  domain: video
  params: [vid]          # 必携参数（继承给子元素）
  children: [play_btn, like_btn, cp_avatar, share_btn]
  owner: client-team

# pages.yaml 条目
- pgid: video_detail
  desc: 视频详情/播放页
  screenshot: /admin/screenshots/video_detail.png   # 可选：页面截图 URL，管理台登记时预览
  domain: video
  params: [vid]
  elements: [play_btn, like_btn, episode_item]   # 页面结构体：本页允许出现的元素（含继承链叶子）
  owner: client-team

# biz_events.yaml 条目
- code: bet_placed
  desc: 投注成功（事实）
  domain: bet
  source: server         # client | server
  params: [bet_id, match_id, market_type, odds, stake_amt, currency]
  owner: backend-team
```

### 4.1 页面结构体（elements 绑定）

页面可选声明 `elements: [eid...]`，即「这个页面上允许出现哪些元素」：

- **声明后强校验**：该页面上报了未绑定的元素 → 网关判 `error` 进隔离区（典型 bug：复用组件忘了改 pgid、元素埋错页面）
- **不声明不校验**：迁移期兼容，新页面登记时建议必填
- CI 与管理台都会校验绑定引用的 eid 必须已在元素池登记
- 对应大同「页面结构体」：结构即约束，错误结构进不了仓

### 4.2 登记生命周期（status）

所有条目可选 `status` 字段，跟踪登记条目从提出到上线的状态：

| status | 含义 |
|---|---|
| `draft` | 登记中（口径还在讨论） |
| `dev` | 已合并，终端开发中 |
| `testing` | 联调验证中（配合管理台「实时联调」+ 断言 API） |
| `live` | 线上运行（**缺省值**） |
| `deprecated` | 已废弃（保留登记避免老版本判未登记，新代码禁止引用） |

状态流转：管理台 **发布/废弃** 或 PR 修改；生产审计在 DB + 管理台。
