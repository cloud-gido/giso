# SDK 1.0.10 心跳修复说明（1.0.9 → 1.0.10）

> 1.0.9 已发布且不可覆盖。1.0.10 修正应用心跳口径；接入配置、App Key、endpoint 不变，只改依赖版本号。

## 坐标

| 端 | 1.0.10 |
|---|---|
| Android | `com.giso:tracker:1.0.10` |
| Web | `@cloud-gido/giso-tracker-web@1.0.10` |
| iOS | SwiftPM tag `v1.0.10`，product `GISOTracker` |
| Flutter | `giso_tracker` @ tag `v1.0.10`，path `sdk/flutter/giso_tracker` |

## 修复内容

- `app_heartbeat` 是应用级事件，只携带 `page.fg_dur`，不继承 `pgid` / 页面参数。
- 前台每 60s（远程可配置）上报；退后台补最后不足一个周期的区间，然后停止。
- `app_background` 同样只携带应用级 `fg_dur`。
- 远程心跳间隔返回后立即重排定时器，避免本地测试间隔造成首跳提前。
- 管理台事件筛选补充 `app_heartbeat`。

## 验证

1. 重新打开 App，确认 `common.sdk_vrsn = 1.0.10`。
2. 前台约 70s，应见一条 `app_heartbeat`，只有 `page.fg_dur ≈ 60000`。
3. 再等待一段不足 60s 后退后台，应先见一条短区间 `app_heartbeat`，再见 `app_background`。
4. 心跳不应因当前页面未登记而报错。
