# 四条业务线埋点设计

> 所有页面/元素/事件均已在 `schema/` 登记。本文档说明每条业务线的埋点结构、关键漏斗与指标口径。
> 通用规律：**列表页看曝光点击（CTR），详情页看转化（biz_event），资金动作看服务端事实流。**

---

## 1. 长视频

### 页面结构与参数继承

```
video_feed (推荐流, pg_params: tab_name)
 └─ video_card (vid, pos, rec_trace_id)
     ├─ play_btn          → 继承 vid, pos, rec_trace_id
     ├─ like_btn          → 继承 vid, pos, rec_trace_id
     ├─ cp_avatar (cp_id) → 继承 + cp_id
     └─ share_btn         → 继承 vid, pos, rec_trace_id

video_detail (pg_params: vid, series_id, ep_num)
 ├─ episode_item (ep_num, vid)
 ├─ fullscreen_btn
 └─ like_btn / share_btn   → 继承页面 vid
```

### 播放链路（biz_event）

```
element_click(play_btn) ─► video_play_start ─► video_play_heartbeat ×N ─► video_play_end
                                  │ 失败
                                  └─► video_play_error
```

### 核心指标口径

| 指标 | 口径 |
|---|---|
| 卡片 CTR | `element_click(video_card)` / `element_exposure(video_card)`，同 pgid 同 pos 维度 |
| 播放成功率 | `video_play_start` / `element_click(play_btn)` |
| 人均播放时长 | sum(heartbeat.play_dur) / 播放 UV（不用 play_end 单条总时长，防丢报） |
| 完播率 | `video_play_end` 中 `play_pos / video_dur ≥ 0.9` 的占比 |
| 推荐效果 | 按 `rec_trace_id` 关联推荐请求与曝光/点击/播放 |

## 2. 体育博彩

### 页面结构

```
bet_lobby (sport_id, tab_name)
 └─ match_card (match_id, sport_id, pos, is_live)
     └─ odds_btn (market_type, selection_id, odds)  → 继承 match_id, is_live

bet_match_detail (match_id, sport_id, league_id, is_live)
 └─ odds_btn (market_type, selection_id, odds)      → 继承页面 match_id

bet_slip (bet_type)
 └─ bet_confirm_btn (bet_type, stake_amt, currency)
```

### 投注漏斗（客户端意图 + 服务端事实）

```
element_exposure(match_card)            曝光
  ─► element_click(odds_btn)            选盘
  ─► bet_slip_add                       加入注单
  ─► element_click(bet_confirm_btn)     确认
  ─► bet_submit          [client]       提交（生成 bet_id）
  ─► bet_placed          [server]       成功 ──► bet_settled [server] 结算
        └► bet_rejected  [server]       被拒（fail_reason）
```

### 核心指标口径

| 指标 | 口径 |
|---|---|
| 赛事卡 CTR | click/exposure(match_card)，分 is_live 维度 |
| 投注转化率 | `bet_placed`(server) UV / `element_exposure(match_card)` UV |
| 注单提交成功率 | `bet_placed` / `bet_submit`，按 `bet_id` 关联；差值拆 `bet_rejected.fail_reason` |
| 流水 / GGR | **仅用服务端流**：sum(`bet_placed.stake_amt`)；GGR = 流水 − sum(`bet_settled.payout_amt`) |
| 端到端对账 | `bet_submit` 与 `bet_placed`+`bet_rejected` 按 bet_id 日对账，差异率 >1% 告警 |

> 合规注意：投注相关事件在网关侧写入 `geo_region`；客户端不采 GPS；负责任博彩功能（限额设置、自我排除）的元素点击同样登记埋点，监管审计需要。

## 3. 预测市场

### 页面结构

```
pm_lobby (pm_category, tab_name)
 └─ market_card (pm_market_id, pm_category, pos)
     └─ outcome_btn (outcome_id, price)   → 继承 pm_market_id

pm_market_detail (pm_market_id, pm_category)
 ├─ outcome_btn (outcome_id, price)
 └─ order_confirm_btn (order_side, price, shares)
```

### 交易漏斗

```
element_exposure(market_card)
  ─► element_click(outcome_btn)
  ─► element_click(order_confirm_btn)
  ─► pm_order_submit     [client]
  ─► pm_order_filled     [server] ──► pm_market_resolved [server]
        └► pm_order_cancelled [server]
```

### 核心指标口径

| 指标 | 口径 |
|---|---|
| 市场卡 CTR | click/exposure(market_card)，分 pm_category |
| 下单转化率 | `pm_order_filled` UV / `element_exposure(market_card)` UV |
| 成交率 | `pm_order_filled` / `pm_order_submit`（限价单可能不成交，分 order_side） |
| 交易量 | sum(`pm_order_filled.price × shares`)，仅服务端流 |
| 市场热度 | 详情页 `page_enter` UV + 订单簿区域元素曝光（用于发现「看而不买」的市场） |

## 4. 资讯

### 页面结构

```
news_feed (news_cat, tab_name)
 └─ article_card (aid, news_cat, pos, rec_trace_id)
     └─ share_btn   → 继承 aid

news_article (aid, news_cat)
 ├─ related_article (aid, pos)   # 注意：此 aid 是被推荐文章的，覆盖页面继承值
 └─ share_btn
```

> 参数覆盖规则：子元素自有参数与继承参数同 key 时，**自有值优先**（如 related_article 的 aid）。

### 阅读结算（biz_event）

`news_read` 在离开文章页时一次性上报（`read_dur` 累计前台阅读毫秒、`read_pct` 最大滚动深度），比纯 `pg_stay` 更准——停留 ≠ 阅读。

### 核心指标口径

| 指标 | 口径 |
|---|---|
| 文章 CTR | click/exposure(article_card)，分 news_cat、pos |
| 有效阅读率 | `news_read` 中 `read_dur ≥ 15s 或 read_pct ≥ 0.5` 占比 |
| 相关推荐效果 | click/exposure(related_article) |
| 导流价值 | 资讯页 → 博彩/预测市场的跨域归因：下游页面 `ref_pgid = news_article` 的 page_enter 占比 |

---

## 5. 跨业务线全局漏斗

四条业务线共享账号与钱包，全局关键链路用 `ref_pgid / ref_eid` 归因链 + 服务端事实流串联：

```
获客:  app_install → signup_completed[server]      （分 channel）
变现:  page_enter(deposit) → deposit_completed[server]   充值转化
跨域:  video/news 内容消费 → bet/pm 交易转化        （ref_pgid 归因）
留存:  app_launch 去重 → 次留/7留，分首次触达业务线
```

北极星建议：**充值用户的周交易频次**（bet_placed + pm_order_filled 合并），内容线（video/news）的考核口径是其对交易线的导流转化贡献。
