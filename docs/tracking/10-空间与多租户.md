# 空间与多租户

GISO 支持**公司级多空间**隔离埋点注册表与联调数据，典型空间：`default`（默认）、`longvideo`（长视频）、`sports`（体育）。

## 角色

| 层级 | 角色 | 能力 |
|------|------|------|
| 平台 | `system_admin` | 创建空间、管理全局账号、访问所有空间 |
| 平台 | `user` | 无全局特权，权限由 `space_members` 决定 |
| 空间 | `space_admin` | 本空间注册表审批/发布、成员与 App Key |
| 空间 | `editor` | 提交待审批登记 |
| 空间 | `viewer` | 联调、统计只读 |

默认管理员 `admin` / `admin123` 为 `system_admin`。

## 数据隔离

- 管理 API 通过请求头 **`X-GISO-Space`**（或 query `space`）指定当前空间。
- 注册表 PostgreSQL 主键：`(space_key, kind, entry_key)`。
- 上报：`X-App-Key` → 空间；网关写入 **`common.space`**，并按该空间注册表校验。

## App Key 命名（Phase 0 前缀规则）

未在 `space_app_keys` 显式绑定时：

| 前缀 | 空间 |
|------|------|
| `video-*` / `longvideo-*` | `longvideo` |
| `sports-*` / `sport-*` | `sports` |
| 其他 | `default` |

## 数据库迁移顺序

结构迁移由 `DbMigrator` 管理，**每条仅执行一次**，严格顺序：

| 版本 | 脚本 | 依赖 |
|------|------|------|
| V1 | `V1__registry.sql` | — |
| V2 | `V2__admin_users.sql` | V1 |
| V3 | `V3__approval.sql` | V2（含 editor 角色） |
| V4 | `V4__spaces.sql` | — |
| V5 | `V5__registry_space.sql` | V4（spaces 表）、V3 |

版本记录在 `schema_migrations` 表。旧库首次升级会自动 **backfill** 已存在的对象，避免 V3/V5 重复跑。

**数据同步**（非版本化）：`sync_longvideo_registry.sql` 在 **V5 完成 + YAML bootstrap 写入 default 之后** 执行，将注册表复制到 `longvideo`。

Doppler `INFRA_GISO_APP_KEYS` 同步入库时自动按前缀绑定空间。

## 上报与 Kafka（不会双写）

每条 `POST /v1/track` 请求：

1. 读取 `X-App-Key`（或 query `k`）
2. 解析为**一个** `space_key`（`resolveSpaceForAppKey`）
3. 注入 `common.space`
4. 用**该空间**注册表校验
5. **写一次** Kafka（`giso_events_raw` / `_test` / `quarantine`）

Kafka **不按空间分 Topic**；分析侧用 `common.space` / Doris `space_key` 列过滤。

```
video-android-prod  →  longvideo  →  1 条 Kafka 消息（common.space=longvideo）
demo-key            →  default    →  1 条 Kafka 消息（common.space=default）
```

## 注册表镜像 ≠ 事件双写

| 类型 | 是否复制到多空间 | 说明 |
|------|------------------|------|
| **注册表配置**（params/pages/elements/events） | 会 | 启动时 `sync_longvideo_registry.sql`；在 **default** 空间增删改时 mirror 到 `longvideo` |
| **埋点事件**（track） | 不会 | 严格按 App Key 进一个空间 |

因此在管理台切换 `default` / `longvideo` 可能看到**相近的登记配置**；联调里应看每条事件的 `common.space` 或工具栏 **space:** 标签。

## 管理台联调与 SSE

**SSE（Server-Sent Events）**：管理台「实时联调」页与 Gateway 建立的 **HTTP 长连接**，服务端把新校验结果 **推** 到浏览器。仅用于运维联调，**App 不上报 SSE**。

- 连接：`GET /admin/api/stream?space={当前空间}`
- 按空间隔离推送（`SseHub` 只推给订阅同一 `space` 的浏览器）

**切换空间**（顶栏空间切换器）时：

1. 断开旧 SSE
2. 按新空间重载 `/admin/api/events` 缓冲
3. 重建 SSE

进入「实时联调」页时同样会按当前 `X-GISO-Space` 刷新。避免误把上一空间事件当成当前空间。

## 管理台

- 顶栏**空间切换器**切换当前工作空间。
- **空间管理**：成员、App Key；平台管理员可创建新空间。
- **账号管理**：仅维护平台 `system_admin` / `user`；空间角色在「空间管理」配置。

## Doris

`ods_events.space_key` 来自 `common.space`；存量集群执行 `server/doris/05_alter_add_space.sql` 并更新 Routine Load。
