# SDK 接入指南（业务方视角）

> 你是某条业务线的开发/产品，要新增一批埋点。照本指南六步走完即接入完成。
> 核心规则一句话：**先登记、用常量、口径交给 SDK、资金走服务端。**

---

## 第 0 步：明确数据需求

接入不是从写代码开始，是从「要回答什么问题」开始。在任务系统建单，写清楚：

- 要看什么指标（如：新功能入口的 CTR、某漏斗的转化率）
- 维度怎么拆（按页面？按位置？按版本？）

> 经验：说不清楚看数方式的埋点需求，登记阶段会被打回。

## 第 1 步：登记（唯一新增入口）

所有页面 / 元素 / 参数 / 业务事件 **先登记后使用**。两种方式：

| 方式 | 适用 |
|---|---|
| 改 `schema/*.yaml` 提 PR | 标准路径，PR 描述关联任务单，数据负责人 + 终端负责人双 CR |
| 管理控制台「注册表配置」页直接增改 | 联调期快速迭代；改动写回 YAML 后**仍需提交 Git 评审** |

登记时遵守命名规范（详见 `03-命名与登记规范.md`）：

```
页面 pgid   {业务域}_{页面}        bet_match_detail / news_article
元素 eid    {名词}_{类型后缀}      like_btn / market_card（全局唯一，不绑定页面！）
参数 key    snake_case ≤32       vid / stake_amt（金额=最小货币单位整数+currency）
业务事件    {域}_{对象}_{动作}     bet_placed / video_play_start
```

判断要不要新增：

- 能用「曝光/点击 + 已有元素 + 参数」表达的，**不要新增业务事件**
- 同语义元素已存在的（如别的页面已有 `share_btn`），直接复用，页面差异由 `pgid` 体现

## 第 2 步：拿常量

登记合并后 CI 自动生成三端常量，禁止手写字符串：

```
Web      sdk/web/src/generated.ts          import { Pages, Elements, Params, BizEvents }
Android  com.giso.tracker.Pages/Elements/Params/BizEvents
iOS      GISOTracker 包内 Generated.swift
```

本地立即要用：`python3 tools/codegen/generate.py` 手动生成。

## 第 3 步：集成 SDK（一次性）

应用已接过的可跳过。初始化放应用入口，`endpoint` 指向网关：

```ts
// Web
const tracker = Tracker.init({ appId: 'giso', appVersion: APP_VERSION,
  endpoint: 'https://t.example.com/v1/track', debug: isDev });
```

```java
// Android (Application.onCreate)
Tracker.init(this, TrackerConfig.builder("giso", BuildConfig.VERSION_NAME,
    "https://t.example.com/v1/track").debug(BuildConfig.DEBUG).build());
```

```swift
// iOS (App 启动)
var config = TrackerConfig(appId: "giso", appVersion: version,
                           endpoint: URL(string: "https://t.example.com/v1/track")!)
config.debug = isDebug
Tracker.initialize(config: config)
```

登录/登出时同步 `setUid` / `clearUid`。生命周期事件（激活/启动/前后台）SDK 自动报，不用管。

## 第 4 步：开发埋点

### 页面：进出各一行

在页面生命周期钩子（路由切换 / onResume / viewDidAppear）调用，**建议在基类收口一次，业务页面只声明 pgid 和参数**：

```ts
tracker.enterPage(Pages.BET_MATCH_DETAIL, { [Params.MATCH_ID]: matchId, [Params.IS_LIVE]: 1 });
// 离开时 exitPage()，pg_stay 自动计算
```

### 元素：bind 即埋完，曝光点击全自动

```ts
// 容器 bind 一次参数，子元素自动继承（这是少写 80% 代码的关键）
tracker.bind(cardEl, { eid: Elements.MATCH_CARD, pos: index,
  params: { [Params.MATCH_ID]: m.id, [Params.SPORT_ID]: m.sportId } });
tracker.bind(oddsBtnEl, { eid: Elements.ODDS_BTN,
  params: { [Params.MARKET_TYPE]: '1x2', [Params.SELECTION_ID]: s.id, [Params.ODDS]: s.odds } });
// odds_btn 的曝光/点击自动带上 match_id、sport_id，无需重复填
```

**不要做的事**：

- 不要自己监听 scroll/触摸去判断曝光——口径（可视≥50% 持续≥500ms）由 SDK 统一，自己判会造成口径分裂
- 不要在接口回调里报「点击」——点击=手势完成；「操作成功」用业务事件表达

### 业务事件：仅限曝光/点击表达不了的

```ts
tracker.bizEvent(BizEvents.VIDEO_PLAY_START, { [Params.VID]: vid, [Params.IS_AUTO]: 0 });
```

**资金红线**：投注成功、订单成交、充值到账等资金/结算事实，**必须由后端服务上报**（`source: server`），客户端最多报「意图」（如 `bet_submit`）。客户端报的钱不作数——网关层强制：端上上报 `source: server` 的事件直接判 error 进隔离区。

后端服务接入 `sdk/server`（`giso-server-sdk`），事务提交后直写 Kafka：

```java
ServerReporter reporter = new ServerReporter("kafka:9092", "giso_events_raw", "giso");
reporter.report(uid, BizEvents.BET_PLACED, Map.of("bet_id", betId, ...), null);
```

## 第 5 步：联调自测

1. SDK 开 `debug: true`（不攒批，实时上报）
2. 打开管理控制台 → **实时联调**，用自己的 did 过滤
3. 操作一遍功能，逐条核对：

| 标记 | 含义 | 处理 |
|---|---|---|
| 绿·正常 | 通过 | — |
| 黄·缺失 | 必携参数为空 | 数据已入库但带缺陷标记，**必须修** |
| 红·错误 | 未登记/类型错误/元素未绑定页面 | **数据进隔离区，不入库**，回去登记或改类型 |

4.（可选）用断言 API 把联调用例固化：声明这条链路应该产生的事件序列，跑一遍功能后一把比对：

```bash
curl -X POST $GATEWAY/admin/api/assert -d '{
  "did": "你的did",
  "expect": [
    { "event": "page_enter",    "pgid": "video_feed" },
    { "event": "element_click", "eid":  "video_card" },
    { "event": "biz_event",     "code": "video_play_start" }
  ]
}'   # → pass=true 即链路完整；detail 里逐条标命中/缺失
```

4. 核对参数值是否符合登记的取值规则（如 `pos` 是否 0-based、金额是不是分）

## 第 6 步：验收上线

1. 测试按需求单过一遍上报测试
2. 数据负责人在「质量统计」页验收（量级合理、缺失率/错误率达标）
3. **验收不通过，需求不随版本上线**（流程卡点）
4. 上线后 24h 关注新版本上报质量，异常率 >1% 会触发告警

---

## 红线清单（Review 必查）

1. 手写字符串 eid/pgid/参数 key —— 必须用生成常量
2. 自造事件名（`xxx_click` 这类）—— 只有 9 个标准事件，业务差异用元素+参数表达
3. 客户端上报资金事实 —— 资金/结算只认服务端流
4. 子元素重复填父级参数 —— 用参数继承，重复填写容易出现两处不一致
5. 自定义曝光/点击判定 —— 口径属于 SDK，需要调整找数据负责人改远程配置（`/v1/config` 下发，不用发版）
6. 上报中携带 PII（手机号/证件/银行卡）—— 一律禁止，用户标识只允许 `uid`/`did`
7. 把页面编进元素名（`home_video_card`）—— 元素全局唯一，页面差异由 `pgid` 表达
