# 账号与权限体系

> 对齐 **GIDO** 三层防护思路；含 **审批流**（编辑员提交 → 管理员批准）。  
> 运维配置见 [deploy/DEPLOYMENT.md](../../deploy/DEPLOYMENT.md)；登记规范见 [03-命名与登记规范](03-命名与登记规范.md)。

---

## 一、与 GIDO 对照

| 层级 | GIDO | GISO |
|------|------|------|
| **网络** | Ingress IP 白名单 | `/admin/` 内网白名单；`/v1/track` 公网 |
| **应用登录** | 前端 + JWT + PG 用户表 | HTTP Basic + `admin_users`（BCrypt） |
| **变更治理** | PR / 工作流 | **pending 审批** + 可选 Git YAML PR |
| **数据面** | 业务 Token | `X-App-Key`（与登录无关） |

---

## 二、角色与权限

| 角色 | 注册表 | 审批 | 账号管理 | 联调/统计 |
|------|--------|------|----------|-----------|
| **admin** | 增删改、发布、废弃 | 批准 / 驳回 | 创建 / 禁用 | 全部 |
| **editor** | 新增/改 **待审批** 条目 | — | — | 查看 + 断言 |
| **viewer** | 只读 | — | — | 查看 + 断言 |

### 审批流（核心）

```
编辑员保存登记 ──► status=pending（不参与线上校验）
                        │
                        ▼
              管理员「批准」──► live（全集群秒级生效，参与 /v1/track）
              管理员「驳回」──► draft（退回修改）
```

- `pending` 与 `draft/dev` 一样：**不校验** App 上报（未登记仍进隔离区）。
- 仅 `live` / `testing` 参与校验（与现网一致）。

### API

| 端点 | 角色 |
|------|------|
| `GET /admin/api/me` | 全部（含 `pending_count`） |
| `GET /admin/api/registry/pending` | 全部（编辑员只读收件箱说明） |
| `POST /admin/api/registry/{kind}/approve?key=` | admin |
| `POST /admin/api/registry/{kind}/reject?key=` | admin |
| `POST /admin/api/registry/{kind}` | admin / editor |
| `GET/POST/PUT/DELETE /admin/api/users` | admin |

## 三、三层防护（网络 + 登录 + App Key）

### L3 · App Key

见 [08-接入常见问题FAQ](08-接入常见问题FAQ.md) Q5。

### L2 · 管理台账号

**Doppler 种子（首次空库 bootstrap）：**

| Doppler | K8s |
|---------|-----|
| `INFRA_GISO_ADMIN_USER` / `PASSWORD` | `GISO_ADMIN_*` |
| `INFRA_GISO_VIEWER_USER` / `PASSWORD` | `GISO_VIEWER_*` |
| `INFRA_GISO_ADMIN_USERS` | `GISO_ADMIN_USERS`（`user:pass:role`） |

**管理台创建编辑员示例（admin 登录 → 账号管理）：**

- 用户名：`pm_zhang`
- 角色：**编辑员**
- 用途：登记参数/页面，提交待审批

持久化：`giso.admin_users`（迁移 `V2` + `V3` 含 `editor` 角色）。

### L1 · Ingress 白名单

生产环境 `/admin/` 仅内网可达，与 Basic Auth 叠加。
