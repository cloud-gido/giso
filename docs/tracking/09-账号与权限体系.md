## 一、与 GIDO 对照

| 层级 | GIDO | GISO |
|------|------|------|
| **网络** | Ingress IP 白名单 | `/admin/` 内网白名单；`/v1/track` 公网 |
| **应用登录** | 前端 + JWT + PG 用户表 | Cookie 会话 + `admin_users`（BCrypt） |
| **变更治理** | PR / 工作流 | **pending 审批** + 可选 Git YAML PR |
| **数据面** | 业务 Token | `X-App-Key`（与登录无关） |

## 二、两层 RBAC 模型

### 平台角色（`admin_users.role`）

| 值 | 含义 |
|----|------|
| `system_admin` | 平台管理员：全部空间、账号管理、系统设置 |
| `user` | 平台用户：无全局特权，**必须**在 `space_members` 中有记录才能登录控制台 |

### 空间角色（`space_members.role`）

| 值 | 注册表 | 审批 | 成员/App Key | 联调/统计 |
|----|--------|------|--------------|-----------|
| `space_admin` | 本空间增删改发布 | 批准/驳回 | 管理 | 本空间 |
| `editor` | 提交 **pending** | — | — | 本空间 |
| `viewer` | 只读 | — | — | 本空间 |

平台 `system_admin` 隐式拥有所有 active 空间的 `space_admin` 权限（无需 `space_members` 行）。

**规范化入口（代码）**：`RbacRoles` · 授权矩阵 `AdminPermissions` · 创建后授权 `RbacProvisioning`。

### 审批流（核心）

```
编辑员保存登记 ──► status=pending（不参与线上校验）
                        │
                        ▼
              空间管理员「批准」──► live（全集群秒级生效）
              空间管理员「驳回」──► draft
```

## 三、账号生命周期

### 1. 种子账号（Doppler / 环境变量）

| Doppler | K8s | 行为 |
|---------|-----|------|
| `INFRA_GISO_ADMIN_USER` / `PASSWORD` | `GISO_ADMIN_*` | 平台管理员；启动时若被误降为 `user` 会恢复 |
| `INFRA_GISO_VIEWER_USER` / `PASSWORD` | `GISO_VIEWER_*` | 平台 `user` + 自动加入**全部** active 空间（`viewer`） |
| `INFRA_GISO_ADMIN_USERS` | `GISO_ADMIN_USERS` | `user:pass:role` 逗号分隔；`editor`/`viewer` 同上自动加空间 |

空库首次启动：若无 Doppler，写入默认 `admin` / `admin123` / `system_admin`。

### 2. 管理台创建（推荐）

**平台管理员 → 账号管理 → 新增账号**：

1. 选择 **平台用户** 或 **平台管理员**
2. 平台用户须指定 **空间角色**（默认 `viewer`）与 **目标空间**（或「全部空间」）
3. 创建后自动写入 `space_members`，用户可立即登录

无需再手动去「空间 → 成员」补一步（仍可后续调整成员）。

### 3. 登录准入

- 密码正确 + 会话建立前检查：非 `system_admin` 须至少属于一个 active 空间
- 否则返回 `403` / `code=no_space_membership`，**不会**进入控制台死循环
- `/me` 在当前空间无权限时，自动回退到用户第一个可访问空间

### 4. 治理护栏

- 不能将**最后一个**平台管理员降级为 `user` 或禁用
- 仅 **Doppler 种子**中的平台管理员在启动时自动恢复 `system_admin`（UI 创建的账号不受此影响）
- 创建新空间时，创建者自动成为该空间 `space_admin`

## 四、API 与角色

| 端点 | 最低权限 |
|------|----------|
| `POST /admin/api/login` | 公开（须有效账号 + 空间准入） |
| `GET /admin/api/me` | 已登录 |
| `GET /admin/api/registry/pending` | 本空间任意成员 |
| `POST .../approve` / `reject` | `system_admin` 或 `space_admin` |
| `POST /admin/api/registry/{kind}` | `system_admin` / `space_admin` / `editor` |
| `GET/POST/PUT/DELETE /admin/api/users` | `system_admin` |
| `POST /admin/api/users` body | 可选 `space_key`、`space_role`、`all_spaces` |
