# SDK 1.0.11 修复说明（Android 退后台合规）

> **必升**：使用 Android / Flutter-Android 且出现切后台崩溃（`NetworkOnMainThreadException`）的团队。  
> 接入配置、App Key、endpoint 不变，只改依赖版本号。

## 坐标

| 端 | 1.0.11 |
|---|---|
| Android | `com.giso:tracker:1.0.11` |
| Web | `@cloud-gido/giso-tracker-web@1.0.11` |
| iOS | SwiftPM tag `v1.0.11`，product `GISOTracker` |
| Flutter | `giso_tracker` @ tag `v1.0.11`，path `sdk/flutter/giso_tracker` |

## 根因

1.0.9 / 1.0.10 为提高 OEM 退后台送达率，在 `Activity.onStop` **主线程同步 HTTPS flush**，触发 Android 禁止的主线程网络 → `NetworkOnMainThreadException`。

## 业界合规做法（本版）

| 步骤 | 线程 | 行为 |
|---|---|---|
| 入队 `app_heartbeat`（尾区间）+ `app_background` | worker（主线程短等） | 写入内存队列 |
| **本地落盘** | worker（主线程短等，通常 &lt; 几十 ms） | 进程随后被杀也不丢 |
| **HTTPS 上报** | **仅** `giso-tracker` worker | 短持 WakeLock；失败留 spill，下次前台补发 |
| 进前台 flush / spill | worker，**不阻塞**主线程 | 不拖进场动画 |

原则：

- 主线程禁止网络（Android 合规）
- 不长时间阻塞 `onStop` / `onStart`（避免 ANR、不影响 App 功能）
- 先落盘再异步发送（与 Firebase / 主流分析 SDK 一致的可靠性权衡）

## 验证

1. 确认 `common.sdk_vrsn = 1.0.11`
2. 打开播放页后反复 Home / 返回，**不应再崩溃**
3. 管理台仍能看到 `app_background`（及可能的尾区间 `app_heartbeat`）；弱网/杀进程后重启可补发 spill
